Módulo de credenciales de usuario

En la barra lateral se muestra el módulo de credenciales. Hasta el 18/08/2016 he detectado las siguientes condiciones para ese menú:

  • cuando el usuario no se ha firmado y está recuperando contraseña, el menú no aparece o aparece vacío

  • cuando el usuario se ha firmado, pero no ha seleccionado perfil, aparecen solamente el primer li y el último li

  • cuando el usuario se ha firmado y seleccionado un rol, de ahí en adelante las pantallas deben mostrar el módulo de credenciales con los tres incisos

  • El usuario puede ingresar a la pantalla de recuperación cambio de contraseña con rol seleccionado o sin el, por lo cual puede ser que aparezcan los tres elementos de lista o solo dos (el primero y el tercero)

  • NUEVO* todas las etiquetas li en el módulo de credenciales necesitan la clase "clearfix" para corregir el ajuste del contenedor cuando los nombres o roles de usuario son muy largos, es necesario agregar la clase a los tres elementos.

Subnavegación

En el header, la segunda barra es nuestra subnavegación. Los mockups hasta la fecha no especifican que esa subnavegación cambia dependiendo de cada rol. Hasta el 18/08/2016, tengo las siguientes observaciones con respecto a la subnavegación:

  • el área para la subnavegación debe aparecer en la pantalla de selección de rol de usuario, sin embargo debe aparecer vacía.

  • Una vez seleccionado el rol, en la siguiente pantalla deberá mostrar la página inicial de cada perfil y la subnavegación específica para ese perfil

Administración de usuarios

El Mockup fué puesto en el SVN el día 19 de agosto. No especifica el proceso mediante el cual el usuario llega a la pantalla de búsqueda de usuario, por lo mismo no es posible determinar si hacen falta vínculos o accesos. Supongo debe existir una pantalla anterior o un vínculo en la navegación, pero no está reflejado en el mockup.

  • NUEVO* todos los formularios deberán agregar clases (mediante programación) a ciertos elementos de la interfaz para señalar cuando han habido errores u omisiones en el llenado de un formulario, como aparece en la guía de estilo en http://www.gob.mx/guias/demo. Estos estilos aplican para: mostrar el mensaje "este campo es obligatorio", mostrar un outline rojo para los elementos de formulario y mostrar asteriscos rojos en etiqueta

  • El mock up utilizado para la pantalla de búsqueda de usuarios (usuarios1.html), especifica que los resultados de la búsqueda se deben ubicar entre el formulario y los botones de enviar. Su contenedor deberá estar oculto hasta que se el usuario de click en buscar. La pantalla cuenta con una tabla de resultados simulada así como los elementos de interfaz necesarios especificados en el mockup
